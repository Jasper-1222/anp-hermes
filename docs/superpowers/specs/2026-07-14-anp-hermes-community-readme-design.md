# ANP Hermes 社区 README 定位更新设计

日期：2026-07-14

## 背景

`anp-hermes` 已经完成 ANP 协议与 Hermes 智能体运行时交互链路的技术验证。当前根 README 更接近功能清单和插件使用说明，尚未充分解释项目为什么存在、解决什么问题，以及社区成员能够从中体验什么。

本次工作只调整对外说明，不继续升级业务代码。

## 目标

将根 `README.md` 调整为面向所有 ANP 社区成员的项目介绍与体验入口，完整说明：

1. 项目目的：提供 ANP 协议与 Hermes 智能体交互方式的技术验证 demo。
2. 项目目标：以一个真实 Agent runtime 展示 DID WBA 身份、Agent 发现、认证、OpenRPC 与 JSON-RPC 调用链路。
3. 解决的问题：帮助社区理解 ANP 协议如何接入真实智能体，以及 Hermes 如何作为 ANP 服务智能体被发现和调用。
4. 实现方式：通过 Hermes 插件机制零侵入注册 ANP 平台，并把 ANP 请求桥接到 Hermes 消息处理流程。
5. 体验方式：保留现有安装、启动、测试、打包和 E2E 命令，让社区成员可以直接运行和验证。

## README 结构

根 README 按以下顺序组织：

1. 项目标题与一句话定位。
2. 项目目的。
3. 解决什么问题。
4. 交互链路图。
5. 实现方式与当前能力。
6. 快速体验。
7. 发布打包。
8. 测试与验证。
9. 配置与可选 Hermes tool RPC。
10. 仓库结构与设计约束。
11. 社区参与方式。
12. 许可证。

## 表达原则

- 面向 ANP 社区所有成员，而不是针对某位负责人撰写采纳申请。
- 正向介绍项目是什么、为什么做、解决什么问题和怎么实现。
- 使用“技术验证 demo”作为核心定位。
- 不专门强调“不是生产级网关”或罗列项目没有实现的能力。
- 不将 AP2、E2EE、DTR、Portal、Mediator 等范围说明组织成长篇负面清单。
- tool RPC 作为可选能力说明，不作为项目首屏主叙事。
- 安全配置、私钥保护和本地测试开关仍在对应使用段落中准确说明。
- 不夸大能力，不把条件执行的真实 LLM E2E 描述为无条件通过。

## 修改范围

本次只修改：

- `README.md`

本次不修改：

- 插件运行时代码；
- 测试代码；
- `plugins/anp-agent/README.md`；

## 仓库公开策略

继续使用现有 GitHub 仓库，不重新创建仓库。现有提交历史、测试体系、OpenSpec 记录和发布脚本能够展示技术验证的演进与可复现性。

更新 README 并完成公开内容检查后，可以直接把当前仓库链接发送给 ANP 社区负责人和其他社区成员。若 ANP 社区后续决定正式收录、迁移或纳入组织，再根据对方建议处理仓库归属；当前不提前复制或重建仓库。

## 验收标准

- README 首屏明确说明这是 ANP 与 Hermes 交互方式的技术验证 demo。
- 社区成员能从 README 理解项目目的、问题、交互流程和实现方式。
- 现有可执行的安装、启动、测试、打包命令不丢失。
- 不新增或修改业务代码。
- README 中不包含密钥、私有配置或不应公开的本机信息。
- 文档内容与仓库当前实现保持一致。
